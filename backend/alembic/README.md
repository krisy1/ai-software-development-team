# Alembic Migrations

## Quick Start

```bash
# Generate SQL for review (offline, no DB needed)
alembic upgrade head --sql

# Apply migrations to database (requires running PostgreSQL)
alembic upgrade head

# Rollback the last migration
alembic downgrade -1

# Rollback to a specific revision
alembic downgrade 0001_initial_schema

# Check current migration status
alembic current

# View migration history
alembic history

# Create a new migration (autogenerate)
alembic revision --autogenerate -m "description_of_change"
```

## Configuration

The database connection is configured in two places:

| Source | Used For | Default |
|--------|----------|---------|
| `alembic.ini` тЖТ `sqlalchemy.url` | Offline SQL generation (`--sql`) | `postgresql+asyncpg://postgres:postgres@localhost:5432/ai_dev_team` |
| `app/config.py` тЖТ `settings.DATABASE_URL` | Online migration execution | Same as above (read from `.env`) |

## Best Practices

1. **Always review autogenerated SQL** тАФ `alembic upgrade head --sql` before applying
2. **Test downgrades** тАФ `alembic downgrade --sql` to verify rollback
3. **One change per migration** тАФ small, focused migrations are easier to review and rollback
4. **Never edit existing migrations after they've been applied** тАФ create a new migration instead
5. **Include `down_revision`** тАФ ensures migrations are applied in the correct order
